2. github.io作为网页

  1. 配置github本地账户(具体的查Git相关笔记).ssh -T git@github.com有响应就是成功
  2. 修改根目录下_config.yml中的几项
  3. 在Node.js项目(即folder)中安装hexo-deployer-git,这是hexo的git部署插件

    npm install hexo-deployer-git –save
    至此已经成功把网页配置为了github.io

3. configuration

3.1 主题替换为next

  1. 下载next主题
  2. 在_config.yml中修改为theme: next即可
  3. 下载后现在一共有两个config文件:一个是根目录下的/_config.yml称为站点配置文件,另一个一个是/themes/next/_config.yml称为主题配置文件

3.2 修改”站点配置文件”

在_config.yml中可以对网页默认配置和站点配置进行设置,配置目录.比较重要的配置(github配置的忽略):

  • title,author,description:网页基础信息
  • post_asset_folder:启用资源文件夹,可以存.md文件需要的png等

hexo(一)安装

Posted on 2024-06-02 In 工具篇 , hexo | 搭建静态博客

参考:hexo官方Documentation>

1. 安装Node.js

从nodesource.github安装,根据命令行操作即可

2. 安装Git

sudo apt install git-core

3. 安装hexo

  1. npm install hexo

  2. echo ‘PATH=”$PATH:./node_modules/.bin”‘ >> ~/.profile
    这一行的目的是将hexo init <工作文件夹>创建的<工作文件夹>的.bin下的内容(其中有hexo)添加为PATH,这样可以直接用hexo命令,而不用npx hexo.但是实际应用中,我发现好像不用这一行也可以直接用hexo命令
