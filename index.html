hexo(三)写博客&发博客

Posted on 2024-06-02 In 工具篇 , hexo | 搭建静态博客

1. 发文章流程

  1. new
    hexo new [option] title
    建立一个新的title.md文件,如果配置了_config.yml中的default_layout参数则是默认会建立一个title文件夹用于存放title.md需要用到的.png等文件.
  • [options]主要只使用-p 或者 --path参数用来指示路径.
  • 路径:title.md会放在根目录/source/_posts下,这里面存的就是要发出去的文章.可以根据分类在_posts下自由放置在比如”工具篇”的文件夹下,并不影响.
  1. 写文章
  2. generate
    hexo generate 或者 hexo g
    生成静态文件
  3. server
    hexo server 或者 hexo s
    启动本地端口预览,默认情况下，访问网址为:http://localhost:4000/.
    hexo server -p 4001
    可以重设端口为4001,访问网址为:http://localhost:4001/.
  4. deploy
    hexo deploy 或者 hexo d
    把静态文件部署在网页(github.io)上
  5. 其他
    hexo g & hexo d可以连续执行两个命令

2. 写文章

与普通的md文件相比,只是多了一个Front-matter.根据需要写就行了

Q: hexo,部署博客到github上，代码更新在github了，但是访问博客内容却没更新？

  1. hexo clean把本地的差分文件清理一下,再hexo g & hexo d
  2. 等会儿时间,有时候会需要10分钟以内的更新时间
  3. 浏览器缓存问题,用手机浏览器先看一下对不对

2. github.io作为网页

  1. 配置github本地账户(具体的查Git相关笔记).ssh -T git@github.com有响应就是成功
  2. 修改根目录下_config.yml中的几项
  3. 在Node.js项目(即folder)中安装hexo-deployer-git,这是hexo的git部署插件

    npm install hexo-deployer-git –save
    至此已经成功把网页配置为了github.io

3. configuration

3.1 主题替换为next

  1. 下载next主题
  2. 在_config.yml中修改为theme: next即可
  3. 下载后现在一共有两个config文件:一个是根目录下的/_config.yml称为站点配置文件,另一个一个是/themes/next/_config.yml称为主题配置文件

3.2 修改”站点配置文件”

在_config.yml中可以对网页默认配置和站点配置进行设置,配置目录.比较重要的配置(github配置的忽略):

  • title,author,description:网页基础信息
  • post_asset_folder:启用资源文件夹,可以存.md文件需要的png等

3.3 修改”主题配置文件”

  1. 开启”标签页”和”分类页”
  2. 手动新建”标签页”和”分类页”,才能在点击”tags”等时知道进入什么对应的页面
    以”分类页”举例,执行命令hexo new page "categories"建立/categories/index.md,并修改内容为

hexo(一)安装

Posted on 2024-06-02 In 工具篇 , hexo | 搭建静态博客

参考:hexo官方Documentation>

1. 安装Node.js

从nodesource.github安装,根据命令行操作即可

2. 安装Git

sudo apt install git-core

3. 安装hexo

  1. npm install hexo

  2. echo ‘PATH=”$PATH:./node_modules/.bin”‘ >> ~/.profile
    这一行的目的是将hexo init <工作文件夹>创建的<工作文件夹>的.bin下的内容(其中有hexo)添加为PATH,这样可以直接用hexo命令,而不用npx hexo.但是实际应用中,我发现好像不用这一行也可以直接用hexo命令
